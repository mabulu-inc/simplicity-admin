## Description

Tests currently require manually running `docker compose up -d` before `pnpm test`, and test suites share schemas at the describe-block level. This task fixes both:

1. **Automatic Docker lifecycle** — Vitest `globalSetup` starts the Postgres container before any tests run and tears it down after all tests complete. No manual `docker compose up` required.

2. **Per-test database isolation** — Each individual test gets its own database, not just a schema per describe block. Prevents cross-test pollution and enables future parallel execution.

3. **Shared test helper** — A shared helper providing `createTestDb()`/`destroyTestDb()` used in `beforeEach`/`afterEach` across all integration tests.

4. **Environment** — Tests read `DATABASE_URL` from `.env`. The globalSetup ensures the container is healthy before proceeding.
