# T-089: Fix Svelte 5 state_referenced_locally warnings

- **Status**: TODO
- **Milestone**: M6 — Code Quality & Hardening
- **Depends**: T-090
- **PRD Reference**: §4.4 (AC-004b, AC-006a)
- **Complexity**: standard

## Description

`svelte-check --fail-on-warnings` reports 8 `state_referenced_locally` errors across two components:

- **AutoForm.svelte** (line 74): `$state(initFormData(columns, values))` captures initial prop values of `columns` and `values`.
- **DashboardBuilder.svelte** (lines 24-29): `$state(name)`, `$state(slug)`, `$state([...roles])`, `$state(isDefault)`, `$state([...widgets])`, `$state([...layout])` all capture initial prop values.

This violates Svelte 5's reactivity model. The existing tests only render once with static props and don't cover prop-change scenarios.

## Approach

1. **Write tests first** that render each component, then re-render with changed props. Assert the expected behavior.
2. **Fix the components** to satisfy the tests and eliminate all `state_referenced_locally` warnings.
3. **Add `--fail-on-warnings`** to the `typecheck` script in `packages/ui/package.json` so warnings are treated as errors going forward.
4. **Verify** `pnpm check` passes with zero errors and zero warnings.

## Produces

- Updated `packages/ui/src/lib/components/AutoForm.svelte`
- Updated `packages/ui/src/lib/components/DashboardBuilder.svelte`
- Updated `packages/ui/package.json` — `--fail-on-warnings` added to typecheck script
- New or updated tests covering prop reactivity in both components
