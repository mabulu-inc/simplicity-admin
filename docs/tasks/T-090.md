Added test-support as UI devDependency for svelte-check resolution; other packages use vitest aliases to avoid cyclic dependency with db.

## Description

The shared `test-support/` directory is imported via fragile relative paths (`../../../../test-support/test-db.ts`) from multiple packages. This breaks `svelte-check` in the UI package because it follows imports transitively and can't resolve `@simplicity-admin/db` and `@simplicity-admin/core` from outside a package context.

Convert `test-support/` into a proper workspace package `@simplicity-admin/test-support` so that:

- All packages import via `@simplicity-admin/test-support` instead of relative paths
- Module resolution works consistently across `tsc`, `svelte-check`, and `vitest`
- The UI package's tsconfig exclude of `./tests/**` can be removed

## Approach

1. Add `package.json` and `tsconfig.json` to `test-support/`
2. Add it to the pnpm workspace
3. Add `@simplicity-admin/test-support` as a devDependency in packages that use it (db, api, auth, cli, ui)
4. Replace all relative `../../../../test-support/` imports with `@simplicity-admin/test-support`
5. Update `vitest.config.ts` globalSetup paths
6. Remove the `./tests/**` exclude from `packages/ui/tsconfig.json`
7. Verify `pnpm check` passes

## Produces

- `test-support/package.json`
- `test-support/tsconfig.json`
- Updated imports across all packages
- Updated vitest configs
- Cleaned up `packages/ui/tsconfig.json`
