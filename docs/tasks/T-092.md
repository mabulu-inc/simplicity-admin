### GitHub Pages docs site

Standalone Astro + Starlight site in `docs-site/` (not part of the pnpm workspace). Deploy via `.github/workflows/deploy-docs.yml` on pushes to `main` that touch `docs-site/**`.

Site structure — pragmatic and comprehensive, aimed at developers:

- **Getting Started**: Introduction, Quick Start, Configuration
- **Core Concepts**: Architecture (provider pattern, package structure), Schema-as-Truth, RBAC (code ceiling, functional roles, column-level grants)
- **Packages**: One page per package (core, db, auth, api, ui, cli) with purpose, exports, and usage
- **Guides**: Adding a Table, Customizing Auth, Writing Plugins, Embedding in Express/Fastify
- **Reference**: Config options, CLI commands, Environment variables

Content should be derived from `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/specs/*.md`, and the actual source code. Do not invent features that don't exist — document what's built.
