## Description

Replace all hand-written DDL in `packages/db/src/bootstrap.ts` with the `@mabulu-inc/simplicity-schema` programmatic API. The YAML schema files already exist in `packages/db/schema/` (tables, roles, functions, mixins). The task is to wire them up so `bootstrap()` delegates DDL to simplicity-schema instead of executing raw SQL.

## Current state

- `bootstrap(pool, config)` in `packages/db/src/bootstrap.ts` contains ~250 lines of hand-written SQL for creating roles, functions, tables, indexes, triggers, and grants
- `packages/db/schema/` already has YAML definitions for all these objects
- `@mabulu-inc/schema-flow` is installed in node_modules but not listed in `packages/db/package.json` â€” it should be removed and `@mabulu-inc/simplicity-schema` added as an explicit dependency

## Key API details

The `@mabulu-inc/simplicity-schema` package (source at `../simplicity-schema`) exports:

```typescript
import { resolveConfig, runMigrate, createLogger } from '@mabulu-inc/simplicity-schema';

// resolveConfig takes ConfigOverrides, returns SimplicitySchemaConfig
const schemaConfig = resolveConfig({
  connectionString: '...',  // required â€” the PG connection URL
  baseDir: '...',           // path to schema/ directory
  pgSchema: '...',          // target PG schema (default: 'public')
});

const logger = createLogger({ quiet: true });

// runMigrate takes (SimplicitySchemaConfig, Logger), returns ExecuteResult
const result = await runMigrate(schemaConfig, logger);
```

**Connection string bridge**: `ProjectConfig.database` contains the connection URL. Pass it as `connectionString` to `resolveConfig()`.

**Schema directory**: The YAML files are in `packages/db/schema/`. Use `path.resolve(__dirname, '../schema')` (or equivalent) for `baseDir`.

**pgSchema**: Read from `config.systemSchema ?? config.schema ?? 'public'` (same logic as current bootstrap).

## Approach

1. Add `@mabulu-inc/simplicity-schema` as a dependency in `packages/db/package.json` (and remove the stale `@mabulu-inc/schema-flow` from node_modules if present)
2. Rewrite `bootstrap()` to:
   - Call `resolveConfig({ connectionString: config.database, baseDir: <schema dir>, pgSchema: <schema> })`
   - Call `runMigrate(schemaConfig, logger)` to apply all YAML declaratively
   - Keep `seedDefaults()` as-is (bcrypt hashing can't be done in YAML)
   - Wrap simplicity-schema errors in `DatabaseError` with code `DB_005`
3. Update `packages/db/src/provider.ts` â€” `migrate()` calls simplicity-schema directly
4. All references to `schema-flow` in the db package have already been updated to `simplicity-schema`
5. Verify existing tests in `packages/db/tests/bootstrap.test.ts` still pass
