# T-091: GitHub Actions CI/CD pipeline with package publishing

- **Status**: TODO

- **Complexity**: standard

## Description

The project has no CI/CD pipeline. All checks (`pnpm check` — build, lint, typecheck, test) run locally only. There is no gated merge protection and no automated package publishing to GitHub Packages.

Set up GitHub Actions to:

1. Run the full `pnpm check` suite on every PR and push to `main`
2. Publish workspace packages to GitHub Packages (`@mabulu-inc` scope) on pushes to `main` when versions change

## Produces

- `.github/workflows/ci.yml` — runs `pnpm check` on PR and push to main
- `.github/workflows/publish.yml` — publishes packages to GitHub Packages on main
- Tests that the workflow YAML is valid (schema validation or act dry-run)

## AC

- `pnpm check` runs on every pull request and every push to `main`
- PRs cannot merge without passing CI (document branch protection rule to enable)
- On push to `main`, changed packages are published to GitHub Packages under the `@mabulu-inc` scope
- Workflow uses pnpm with caching for fast installs
- PostgreSQL service container is available for integration tests
- Secrets use `GITHUB_TOKEN` (no additional secrets needed for GitHub Packages)
