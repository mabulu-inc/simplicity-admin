# T-094: Default systemSchema to _simplicity instead of public

- **Status**: TODO
- **Milestone**: M6 — Code Quality & Hardening
- **Depends**: none
- **PRD Reference**: §2 (Principle 1: zero-config with excellent defaults)
- **Complexity**: standard

## Description

System tables (users, tenants, memberships, etc.) currently default to the `public` schema, mixing with user application tables. This is a poor default — system tables should be isolated in their own schema so developers see a clean `public` schema containing only their own tables.

Change the default `systemSchema` from `public` to `_simplicity`. The underscore prefix signals "framework-managed, don't touch."

Tables are created during `bootstrap()` which runs at startup via the `dev` command and whenever the CLI applies schema. The schema is created automatically (`CREATE SCHEMA IF NOT EXISTS`), so no manual setup is needed.

### What to change

1. **Config default** — `packages/core/src/config/schema.ts` and/or `packages/db/src/bootstrap.ts` line 24: change fallback from `'public'` to `'_simplicity'`
2. **CLI dev command** — `packages/cli/src/commands/dev.ts` line 75: same fallback change
3. **All tests** that rely on the default systemSchema being `public`
4. **Documentation** — update docs-site pages that reference system schema (architecture, configuration, quick-start)
5. **ARCHITECTURE.md** — update any references to where system tables live

### What NOT to change

- Users who explicitly set `systemSchema` in config should still be able to use any schema they want, including `public`
- The schema YAML files in `packages/db/schema/` don't hardcode a schema — they use whatever `pgSchema` is passed to simplicity-schema

## Produces

- Updated default in config/bootstrap
- Updated tests
- Updated docs (docs-site and ARCHITECTURE.md)

## AC

- `defineConfig({ database: '...' })` with no `systemSchema` results in system tables in `_simplicity`
- `defineConfig({ database: '...', systemSchema: 'custom' })` still works
- Bootstrap creates `_simplicity` schema automatically
- `pnpm check` passes
- Docs-site references updated to reflect `_simplicity` as the default
